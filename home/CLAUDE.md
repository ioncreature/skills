# Глобальные правила для Claude Code

## Git
- **Никогда не делать `git commit` / `git push` без явной команды** от меня.
  Завершение работы, зелёные тесты, прошедший acceptance, формулировки
  вроде «сделай X», «допили Y», «справишься?» — это **не команда коммитить**.
  Просто закончи работу и жди. Команда коммитить — это `/cp`, `/scp`,
  явное «коммить» / «закоммить» / «запушь» / «commit» / «push», или
  прямое упоминание commit/push внутри просьбы.
- Не амендить чужие коммиты и не делать `git push --force` без прямой
  просьбы.
- **Работать в текущей ветке. Не создавать новую ветку без прямой
  команды** — даже если текущая ветка дефолтная (`main` / `master`).
  Никакого авто-branch «на всякий случай». Команда создать ветку — это
  явное «создай ветку» / «новую ветку» / «branch» / «checkout -b» или
  прямая просьба переключиться/завести ветку. Просто работай там, где
  сейчас находишься.

## Временные файлы
- **Все временные файлы и папки клади в `./temp/` в корне репозитория**,
  а не в `/tmp/`. Это касается скриптов, дампов, логов, экспериментов,
  тестовых input/output, любых артефактов на «посмотреть и выкинуть».
- Если работаешь не в git-репозитории — используй `/tmp/` как обычно.
- Если папки `./temp/` нет в репозитории — создай её и убедись, что
  `temp/` есть в `.gitignore` (добавь строку, если её нет).
- Это правило распространяется и на субагентов: при делегировании задач
  через Agent tool передавай это требование в промпт, если агент будет
  создавать временные файлы.
